Agent SDK — Developer Guide

Build agents that send messages, stream responses, mount persistent volumes, and survive sandbox restarts.

Quick start Install Agent params Sending messages Events Resuming Sandbox access Live config MCP & skills Token usage Credentials Config files Cleanup Error handling ApiClient pre_start_commands Full example
Two clients, two personas.
  • agent_sdk.Agent — user. Owns one session. Streams events, sends messages, resumes, exposes agent.sandbox.
  • agent_sdk.ApiClient — operator. Stateless REST wrapper. Manages volumes, sessions, files for other people's sessions. Use it for schedulers, admin tools, hive workspace bootstraps.
Pick by persona, not feature: if your code is the user, use Agent; if your code is acting on behalf of users, use ApiClient.

⚡Quick start

Create an agent, send a message, get a response.

Python
import asyncio
from agent_sdk import Agent

async def main():
    agent = Agent("my-agent", provider="local")
    text = await agent.arun("Say hello in one sentence.")
    print(text)

asyncio.run(main())
Default api_url is the hosted Railway server. Override with the api_url= kwarg or the AGENT_API_URL env variable. Set api_url="http://localhost:7778" to use a local server.

💬Sending messages

Four ways to interact: arun, astream, send, run.

arun — full response as a string (most common)
response = await agent.arun("Write a haiku about the ocean.")
print(response)
astream — stream events as they arrive
async for event in agent.astream("Tell me a long story."):
    print(event, end="", flush=True)  # str(event) returns human-readable text
send — fire and forget, returns rpc_id immediately
rpc_id = await agent.send("Start processing the dataset.")

# Listen for results separately:
async with agent.events() as stream:
    async for event in stream:
        if event.get("rpc_id") == rpc_id and event["type"] == "done":
            break
run — sync wrapper around arun
# Spins up its own asyncio event loop. Useful in scripts and notebooks.
text = agent.run("Say hi", timeout=60)
interrupt — cancel a running turn and inject a new message
# All four methods accept interrupt=True:
await agent.arun("Stop and just summarize what you have so far.", interrupt=True)
await agent.send("Cancel and do this instead.", interrupt=True)
Interrupt is purely client-side: cancel() first, then submit. The server has no per-session prompt queue — at most one session/prompt runs against the ACP child at a time per session.
cancel — drop the running prompt without replacing it
await agent.cancel()  # best-effort; same as POST /sessions/{id}/cancel

📡Events

Structured dicts from astream() and events(). str(event) returns human-readable text.

text
A chunk of assistant text. Field: text.
reasoning
Internal reasoning (Claude extended thinking). Field: text.
tool
A tool call. Fields: tool_name, tool_input, tool_use_id.
tool_result
Tool result. Fields: tool_use_id, result.
usage
Token / cost stats. Fields: inputTokens, outputTokens, totalCostUsd.
done
Turn complete. Last event in a stream. Field: stop_reason (e.g. end_turn, cancelled).
error
Server-side error frame. Raised as PromptError from astream; yielded as a dict from events().
Branching on event type
async for event in agent.astream("Analyze this codebase."):
    if event["type"] == "text":
        print(event["text"], end="")
    elif event["type"] == "tool":
        print(f"\n[calling {event['tool_name']}({event.get('tool_input', {})})]")
    elif event["type"] == "usage":
        print(f"\ntokens: {event.get('inputTokens', 0)} in / {event.get('outputTokens', 0)} out")
    elif event["type"] == "done":
        break
events() — long-lived subscription, all events for the session
async with agent.events() as stream:
    async for ev in stream:
        if ev.get("type") == "error":
            log.error(ev)  # yielded, not raised
        elif ev.get("type") == "done":
            break

🔄Conversation continuity & resuming

Sessions persist in Postgres and survive sandbox death + server restart.

Multi-turn — context accumulates automatically
agent = Agent("assistant", provider="local")

await agent.arun("My name is Alice.")
reply = await agent.arun("What's my name?")
# "Your name is Alice."
Resume from a different process
# First run:
agent = Agent("worker", provider="local")
await agent.arun("Remember 42.")
saved_id = agent.session_id

# Later (different process):
agent = Agent("worker", session_id=saved_id)
reply = await agent.arun("What number?")
# "42"
Sandbox reattach is automatic. The session row stores volume_id and the discriminated sandbox_state JSONB. When the previous sandbox is reaped or killed, the next call routes through SessionPool.get_session() — cold-creates a fresh sandbox that mounts the same volume, and the agent CLI resumes from the on-disk transcript via session/load. No special event surfaces — the SDK handles it transparently.
reset_session — start fresh
agent.reset_session()  # clears session_id, sandbox_ref; next call re-registers

📦Sandbox access

Direct exec/read/write/ls on the agent's sandbox. Bypasses the conversation.

agent = Agent("dev", provider="local", cwd="/tmp")
await agent.arun("Create notes.txt with 'hello world'.")

# Read a file directly
content = await agent.sandbox.read_file("/tmp/notes.txt")

# Run a shell command
result = await agent.sandbox.exec("ls /tmp", timeout=30)
print(result["stdout"])
# also: stderr, exit_code, stdout_truncated, timed_out

# Write a file
await agent.sandbox.write_file("/tmp/config.json", '{"debug": true}')

# List a directory
print(await agent.sandbox.ls("/tmp"))

🎚Live config — configure()

Change model, permission mode, or thinking depth at runtime.

await agent.configure(
    mode="bypassPermissions",        # or "default", "acceptEdits"
    model="claude-sonnet-4-6",
    thought_level="high",             # "off" | "low" | "medium" | "high"
)

🛠MCP servers & skills

Extend the agent's capabilities at registration time. Tool whitelisting is the agent CLI's job — set it via configure(mode=…) or in the underlying agent's own config.

MCP servers — extend with custom tools
agent = Agent(
    "with-mcp", provider="local",
    mcp_servers={
        "my-mcp": {
            "type": "local",
            "command": "node",
            "args": ["path/to/server.js"],
        },
    },
)
Skills — npx-installable agent skill bundles
agent = Agent(
    "with-skills", provider="daytona",
    skills=[
        "rllm-org/hive#staging",
        "vercel-labs/agent-skills",
    ],
)
# Skills are installed inside the sandbox before the supervisor starts.
# For local provider, skill installs run on the host.

🧪extra_options — vendor-specific ACP options

Pass-through to the ACP wrapper's _meta.<vendor>.options channel. For agent_type="claude" this lands in claude-agent-acp's userProvidedOptions and onward into @anthropic-ai/claude-agent-sdk's ClaudeAgentOptions. Session-scoped — set at session/new, immutable for the session's lifetime.

Tool restriction — allow/deny lists honored end-to-end
agent = Agent(
    "reader", agent_type="claude", provider="unix_local",
    extra_options={
        "allowedTools": ["Read", "Glob", "Grep"],
        "disallowedTools": ["Bash", "Edit", "Write"],
        "maxThinkingTokens": 10000,
    },
)
# Verified empirically: with disallowedTools=["Read"...], asking the
# agent to read a file results in a clean refusal, not a fallback Bash.
Schema-enforced JSON output — use jsonSchema, not outputFormat
# Claude code constrains its output to the schema. The JSON streams
# back as agent_message_chunk text — agent.arun() returns it raw,
# json.loads(text) is guaranteed to succeed.
schema = MyResponseModel.model_json_schema()
agent = Agent(
    "schema-bound", agent_type="claude", provider="unix_local",
    extra_options={"jsonSchema": schema},
)
text = await agent.arun("…")
result = json.loads(text)  # schema-validated by the CLI

Why jsonSchema and not outputFormat: {type:"json_schema", schema:…}? The wrapped form routes through a StructuredOutput pseudo-tool inside claude-agent-acp that captures the structured payload as a tool result — never re-emitted as assistant text. extraArgs.json-schema takes the same broken path. Setting the bare jsonSchema field instead causes claude code to stream the JSON as agent_message_chunk events, which is what arun() concatenates. Diagnosed by triplet-comparison on the same prompt; see EXTRA_OPTIONS_PLAN.md.

Known field shapes (subset of ClaudeAgentOptions)
# Field          shape                  notes
allowedTools     string[]               # whitelist; merged with built-in defaults
disallowedTools  string[]               # denylist; merged with ACP's ["AskUserQuestion"]
maxThinkingTokensnumber                 # thinking-token budget
jsonSchema       object                 # JSON Schema; constrains response shape
extraArgs        Record<string,string>  # raw claude-cli flags (escape hatch)

For other agent types (codex, opencode), extra_options is currently dropped with a warning at registration — the vendor's ACP-namespace mapping must be added to api.acp_client._VENDOR_META_NAMESPACE after confirming the wrapper's _meta key. claude is the only entry today.

📊Token usage

Cumulative stats on agent.usage.

agent = Agent("worker", provider="local")
await agent.arun("Write a poem.")
await agent.arun("Write another.")

print(agent.usage.input_tokens)
print(agent.usage.output_tokens)
print(agent.usage.total_tokens)
print(agent.usage.total_cost_usd)
print(agent.usage.call_count)

🔑Per-request credentials

Run a session under a specific user's Claude account.

agent = Agent(
    "per-user", provider="daytona",
    oauth_token=user_oauth_token,        # preferred
    api_key=user_anthropic_key,          # fallback
)
# Resolution order: oauth_token arg > CLAUDE_CODE_OAUTH_TOKEN env
#                  > api_key arg > ANTHROPIC_API_KEY env
Credentials never persist to the agent-sdk database. They travel in the request body, are merged into the supervisor's spawn_env, and are scoped to that sandbox. If you supply OAuth, the server scrubs its own ANTHROPIC_API_KEY from that sandbox so there's no silent fallback.
The SDK refuses to forward credentials to a plain http:// server (other than localhost). Use HTTPS in production.

📂Config files & cloning

Load from JSON or YAML
# configs/coder.json
#   {"agent_type":"claude","model":"haiku","provider":"local","cwd":"/workspace"}

agent = Agent.from_file("configs/coder.json", api_url="http://localhost:7778")
agent2 = Agent.from_config("adhoc", {"agent_type": "claude", "provider": "local"})
Clone — same config, different name
base = Agent("base", provider="local", model="haiku")
workers = [base.clone(f"worker-{i}") for i in range(3)]
results = await asyncio.gather(*[w.arun("Summarize AI.") for w in workers])

🧹Cleanup

Context manager — automatic
async with Agent("worker", provider="local") as agent:
    await agent.arun("Do the task.")
# aclose() runs automatically — server-side reap + http close
Manual
agent = Agent("worker", provider="local")
await agent.arun("Do the task.")
await agent.aclose()  # reaps server-side session, then closes httpx client

# Close only the HTTP client (e.g. you want to reap externally):
await agent._api.close()

⚠️Error handling

AgentNotRegisteredError
Operation called before registration; usually internal.
AgentBusyError
Agent is mid-prompt — wait, queue, or use interrupt=True.
AgentTimeoutError
No response within the configured timeout.
AgentConnectionError
Could not reach the server.
PromptError
Server returned an error frame mid-stream. Has .kind and .data attributes (e.g. sandbox_process_died, timeout).
StreamError
SSE connection dropped before done arrived.
from agent_sdk.errors import AgentBusyError, PromptError, StreamError

try:
    text = await agent.arun("Do something.")
except AgentBusyError:
    print("Busy — retry or use interrupt=True")
except PromptError as e:
    print(f"Agent error: kind={e.kind} msg={e}")
except StreamError as e:
    print(f"Connection lost: {e}")

🖥ApiClient operator

Stateless wrapper over every REST route. Use this for schedulers, admin tools, hive workspace bootstraps — anything that creates and manages other people's sessions.

Basic usage
from agent_sdk import ApiClient

async with ApiClient("http://localhost:7778") as sc:
    session = await sc.create_session(
        provider="local", model="haiku",
        name="worker", agent_type="claude",
        secrets={"CLAUDE_CODE_OAUTH_TOKEN": os.environ["CLAUDE_CODE_OAUTH_TOKEN"]},
    )
    sid = session["session_id"]

    await sc.send_message(sid, "Write hello world in Python.")
    async for chunk in sc.stream_events(sid):
        print(chunk, end="")  # raw SSE bytes — parse however you like

    result = await sc.session_sandbox_exec(sid, "echo hello")
    print(result["stdout"])

    log = await sc.get_session_log(sid, limit=200)
    status = await sc.get_session_status(sid)

Stateless w.r.t. resources. One ApiClient instance is safe across thousands of concurrent operations against unrelated sessions. The only state it owns is one httpx.AsyncClient connection pool plus the bearer header.

🚀pre_start_commands

Shell commands that run inside the sandbox before the supervisor starts. Persisted on the session row and replayed when the sandbox is replaced.

Use pre_start_commands to install CLIs, drop config files, or seed state that has to be ready before the agent boots. The full lifecycle:
  • Initial provision: commands run inside the new sandbox.
  • Type 2 recovery (sandbox replaced — external delete, unrecoverable provider error): commands re-run on the replacement sandbox via the recipe stored on sandbox_state.
  • Type 1 recovery (same sandbox restarted): commands do not re-run — the original side-effects are still on disk.
  • Skill-install commands are re-merged from agent.config.skills at recovery time, so editing skills on a long-lived agent takes effect on the next sandbox replacement.
  • For local provider, pre_start_commands are ignored — there's no sandbox boundary to safely run caller commands in.
via Agent — pass pre_start_commands directly
agent = Agent(
    "deploy-bot", provider="daytona",
    cwd="/home/daytona",
    pre_start_commands=[
        "npm install -g cowsay",
        "echo 'project=foo' > /home/daytona/.env",
    ],
    shared_mounts=["shared-data"],          # RO mount under /mnt/shared/
    secrets={"GITHUB_TOKEN": gh_token},      # merges with auto OAuth/API key
)
await agent.arun("verify environment")
# Commands have already run; cowsay is on PATH inside the sandbox.
via ApiClient — operator persona
async with ApiClient("http://localhost:7778") as sc:
    session = await sc.create_session(
        provider="daytona",
        name="deploy-bot", agent_type="claude",
        cwd="/home/daytona",
        pre_start_commands=["npm install -g cowsay"],
        secrets={"CLAUDE_CODE_OAUTH_TOKEN": os.environ["CLAUDE_CODE_OAUTH_TOKEN"]},
    )
    sid = session["session_id"]

    # Inspect what was persisted:
    s = await sc.get_session(sid)
    print(s["pre_start_commands"])
Replay verification — release + delete the lease, then send a new message
# Release current compute lease (snapshot + drop) — the next call will
# either Type-1 resume from snapshot or Type-2 reprovision if the snapshot
# is stale / the provider sandbox is gone:
async with agent_sdk.ApiClient("http://localhost:7778") as sc:
    await sc.release_session(agent.session_id)

await agent.arun("verify environment")
# If a Type-2 reprovision was triggered, pre_start_commands re-ran first.

🚀Full example

Multi-turn coding session with persistent volume, sandbox verification, and cost tracking.

import asyncio, os
from agent_sdk import Agent

async def main():
    async with Agent(
        "coder",
        provider="daytona",
        model="haiku",
        cwd="/home/daytona",
        pre_start_commands=["npm install -g cowsay"],
        api_url="http://localhost:7778",
    ) as agent:

        # Turn 1
        print(await agent.arun(
            "Write a function that reverses a string. Save it to reverse.py.",
        ))

        # Turn 2 — context preserved
        print(await agent.arun("Now write a unit test for that function."))

        # Verify directly via sandbox — no conversation round-trip
        content = await agent.sandbox.read_file("/home/daytona/reverse.py")
        print("\n--- reverse.py ---\n" + content)

        # pre_start_commands installed cowsay; confirm it's on PATH
        cowsay = await agent.sandbox.exec("which cowsay")
        print("cowsay path:", cowsay["stdout"].strip())

        print(f"\nTotal cost: ${agent.usage.total_cost_usd:.4f}")

asyncio.run(main())
