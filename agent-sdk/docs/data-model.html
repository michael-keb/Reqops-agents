AFE Agent SDK — Data Model

Schema reference. Sessions bind to a durable volume; sandboxes are ephemeral compute leases that mount that volume. Sandbox death doesn't kill the session — a new one gets provisioned with the same volume+subpath, and the CLI resumes from the transcript on disk.

Data Model

Three layers: Postgres on the server (durable agent/sandbox/session records + event log), in-memory SessionState (live runtime), and SQLite on the client (resume hints).

Durable storage, independent of any sandbox. Lifecycle via /volumes CRUD. Volumes hold user data only — the supervisor + ACP bins ship in the agent-sdk image at /opt/agent-sdk/runtime/.
On-volume layout:
  agents/<agent_id>/ — per-agent slice (default HOME): snapshot.tar (HOME contents, refreshed each turn-end) + logs/
  workspaces/<name>/ — opt-in shared HOME, mounted at /home/agent for sessions that set sessions.workspace=<name>. Multiple agents passing the same name share the directory and see each other's writes via the kernel. Rejected on daytona.
  shared/<name>/ — opt-in shared folder, mounted at /mnt/<name> for agents that list it in config.shared_mounts
sessions.sandbox_state — JSONB on the sessions row
Pydantic union
Sandbox identity is not a separate table — it's a discriminated-union JSONB blob on the session row, owned by the in-process SessionPool. The recipe lives here too, so deleting a sandbox can't lose its provisioning fingerprint.
type "daytona" | "docker" | "unix_local" | "modal" | "unknown" discriminator — picks the concrete SandboxState class
sandbox_ref TEXT | null opaque provider ref (Daytona UUID, Docker container id, local PID-as-string). Null between cold-create and first start.
listen_port INTEGER | null port-based providers (docker/local/modal). Null for daytona — URL comes from the signed-preview API.
snapshot_path TEXT | null volume-relative path to the latest filesystem snapshot
snapshot_version INTEGER monotonic version, bumped on each release/snapshot write
recipe { agent_type, dockerfile?, shared_mounts[], root?, pre_start_commands[] } provisioning fingerprint — replayed on Type-2 recovery (replacement sandbox)

AgentConfig
JSON blob
agent_type str claude | codex | opencode | gemini | cline | deepagents | openhands | goose
model str?
mcp_servers dict? name → config
skills list | dict? npx skills sources
mode str? "default" | "plan" | "bypassPermissions" | "acceptEdits" | …
thought_level str? "low" | "medium" | "high" — Claude's "thinking" config_id
Stored as JSON in agents.config column. Pure agent identity — cloneable, no per-invocation state. mode + thought_level are persisted here so cold-recovery (Type-2) replays them via _attach_acp.
Not on AgentConfig:
  cwd → sessions.cwd (JSONL-hash key; per-conversation)
  workspace → sessions.workspace (opt-in shared HOME name; overrides agents/<id>/ with workspaces/<name>/)
  dockerfile + shared_mounts + root + pre_start_commands → sandbox_state.recipe (frozen at provision time)
  env → sessions.env / sessions.secrets

Methods

unsubscribe_session(q)

In-memory runtime state. Persistent SSE reader connects once at session creation and fans out to both session-scoped and rpc-scoped queues. Slow subscribers are kicked via _KICK_SENTINEL. ACP/inner session IDs are ephemeral — recreated on connect, restored via native session/load on resume.
