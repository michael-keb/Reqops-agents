Agent SDK — REST API

Two top-level resources: Volumes (durable storage) and Sessions (conversation, with implicit sandbox lifecycle owned by the in-process SessionPool). There is no /sandboxes resource — sandbox identity lives in sessions.sandbox_state JSONB and is surfaced via GET /sessions/{id}/sandbox.

GET read POST mutate DELETE remove SSE server-sent events

Schema

Four tables. Sandbox identity is not a separate table — it lives in sessions.sandbox_state JSONB.

Mental model: a session binds to a volume (durable). The active compute lease is owned in-process by the SessionPool; sandbox_state.sandbox_ref is the opaque provider sandbox identity persisted on the row so cold-recovery (server restart, replacement sandbox) can pick up where the previous lease left off. The recipe (dockerfile, shared_mounts, root, agent_type, pre_start_commands) lives on the session row, never on the compute itself, so deleting a sandbox can't lose it.

Volumes

Persistent storage. Created once, lives for months. Holds ~/.claude, transcripts, workspace.

/volumes
Resource lifecycle
POST /volumes Create + wait until status = "ready". Body {name, provider}.
GET /volumes List — optional ?provider=daytona filter.
GET /volumes/{id_or_name} Inspect — full row plus resolved provider_ref.
DELETE /volumes/{id_or_name}?force=false Delete — 409 if sessions reference it; force=true cascades.
File operations
GET /volumes/{id_or_name}/files/tree?path=shared/ Browse the volume contents.
GET /volumes/{id_or_name}/files/read?path=… Read any file on the volume (shared/, agents/, …).
GET /volumes/{id_or_name}/files/exists?path=… Existence check.
GET /volumes/{id_or_name}/files/download?path=… Raw bytes (forwards content-disposition).
POST /volumes/{id_or_name}/files/edit Overwrite ({path, content}) or search/replace ({path, old_string, new_string, replace_all?}).
POST /volumes/{id_or_name}/files/upload Binary upload — body {path, content (base64)}.
POST /volumes/{id_or_name}/files/mkdir Create directory — body {path}.
POST /volumes/{id_or_name}/files/delete Delete file or directory — body {path}.
POST /volumes/{id_or_name}/files/rename Rename / move — body {path, new_path, overwrite?}; 409 with {error: "exists"} if overwrite=false.
File ops talk directly to the provider's volume primitives — no live sandbox required to seed data.

Sessions

Conversation state. Compute lease is implicit and owned by the in-process SessionPool.

/sessions
Resource lifecycle
POST /sessions Create — eager by default (provisions sandbox + ACP via SessionPool.cold_create); pass "provision": false for lazy.
GET /sessions List sessions currently leased by the pool.
GET /sessions/{id} Read the row — env, redacted secret keys, sandbox_ref, pre_start_commands.
GET /sessions/{id}/status Runtime status — routes through the pool, brings the session up if hibernated.
GET /sessions/{id}/sandbox Sandbox metadata — {provider, sandbox_ref, status, root, url, marker_path}.
GET /sessions/{id}/log?limit=N Event log (oldest first). Each row carries prompt_id.
POST /sessions/{id}/resume Idempotent pre-warm — body may PATCH env / secrets.
POST /sessions/{id}/release Snapshot to volume + drop the pool's compute lease (idempotent hibernation).
DELETE /sessions/{id} Release lease + delete row. Returns 204 even when missing — safe as a "make sure it's gone" primitive.
Messaging / runtime
POST /sessions/{id}/message Fire-and-forget. Returns {rpc_id, status}; events flow into session_log + any /events subscribers.
SSE /sessions/{id}/message+stream Submit + stream the reply in a single round-trip. Same wire format as /events, scoped to one rpc_id.
SSE /sessions/{id}/events Multi-subscriber SSE — every event broadcast by the SandboxSession, plus heartbeat sentinels.
POST /sessions/{id}/cancel Best-effort session/cancel notification on the live ACP child.
POST /sessions/{id}/config Patch model / mode / thought_level — applied live AND persisted on agents.config for cold-recovery.
POST /sessions/{id}/acp/call Generic ACP passthrough — {method, params, notify?}. Server auto-injects the inner sessionId.
POST /sessions/{id}/sandbox/exec One-shot command in the session's sandbox. Auto-recovers if reaped.
Session-scoped filesystem
GET /sessions/{id}/files/tree Recursive directory tree of the live sandbox.
GET /sessions/{id}/files/read?path=… Read a file.
GET /sessions/{id}/files/download?path=… Raw bytes.
POST /sessions/{id}/files/edit Edit / create — same body shape as the volume edit endpoint.
POST /sessions/{id}/files/upload Binary upload — {path, content (base64)}.
POST /sessions/{id}/files/delete Delete file / directory — {path}.
POST /sessions/{id}/files/rename Rename / move — {path, new_path}.
Recovery: conversation endpoints (/message, /message+stream, /events, /cancel, /config, /acp/call, /sandbox/exec, /files/*) all route through SessionPool.get_session(id). Cold-creates from sandbox_state if no lease exists, or returns the live one if it does. There is no separate Type-1 / Type-2 decision in the route handler — it's internal to SandboxSession.start().

Agents

Mostly unchanged. Delete now cleans up the agent's HOME on any volumes that carried its state.

/agents
POST /agents Register an agent definition (no compute provisioned). Rejects cwd / env / dockerfile / shared_mounts with 400 — those belong on session / sandbox.
GET /agents List.
GET /agents/{id} Inspect.
DELETE /agents/{id} Delete — cascades to sessions via FK.

Health & Admin

ops
GET /health Liveness — {status, sessions, busy_sessions}.
GET /admin/sessions Pool snapshot for the dashboard — every active lease + every supervisor instance.
GET /ui Chat UI (also /ui/dashboard, /ui/files, /ui/volumes).

SSE Wire Format

Both /events and /message+stream emit the same shape.

blocks
event: rpc:<rpc_id> — per-prompt SSE block; the body is the raw ACP JSON-RPC frame (notification, terminal result, or error).
: heartbeat — keep-alive comment lines emitted during idle so nginx / cloudflare / EventSource don't close the connection between prompts.
data: <json> — untagged blocks for non-prompt events (setup chatter, server-emitted error frames). Less common.
{"jsonrpc":"2.0","id":"<rpc_id>","result":{"stopReason":"end_turn|cancelled"}} — terminal frame for a prompt.
